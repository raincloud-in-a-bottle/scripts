# Scripts

🌍 **公开脚本仓库** - 存放可分享的脚本工具

## 📝 命名规范

**格式：** `YYYY-MM-DD_<script_name>`

**例子：**
- `2026-03-09_backup.sh`
- `2026-03-10_data_processor/`
- `2026-03-11_web_scraper.py`

## 🎯 使用场景

- 可复用的工具脚本
- 想开源分享的项目
- 不包含敏感信息的脚本
- 对他人可能有用的自动化脚本

## 📁 目录结构

```
scripts/
├── README.md
├── 2026-03-09_backup.sh
├── 2026-03-10_data_processor/
│   ├── main.py
│   ├── README.md
│   └── requirements.txt
└── ...
```

## 📄 许可证

默认使用 MIT 许可证，除非另有说明。

## 💡 提示

- 每次写新脚本前，确认是否应该用公开仓库
- 使用日期前缀方便查找和排序
- 复杂的脚本应该在子目录中包含自己的 README
- 记得移除敏感信息（API keys、密码等）
